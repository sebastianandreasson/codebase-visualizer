Dynamic LOC sizing needs a layout footprint contract

This report uses the current workspace snapshot and the app's own layout/rendering code to show why custom folders and semantic clusters break after LOC-based node sizing, then proposes fixes that preserve prominence without letting large symbols damage the canvas.

Root Cause

There are now two different concepts of size. Layouts still store positions and often use the default symbol footprint of roughly 240 x 108. Rendering then inflates symbols from LOC, zoom, label width, badges, and cluster state. Group folders are derived before that final render footprint is known, so the folders can be smaller than their children.

What This Means

The fastest visible improvement is to compute folder bounds from rendered dimensions. The complete fix is to make every layout strategy consume the same footprint model that rendering uses, then run a final collision pass after dynamic dimensions are known.

src/visualizer/flowModel.ts: group containers use placement/default sizes.
src/visualizer/flowModel.ts: symbol rendering uses zoom-aware LOC dimensions.
src/semantic/projection/refinement.ts: semantic packing already has a footprint input, but it is not the same as final React Flow dimensions.
src/layouts/symbolLayout.ts: default symbol layout allocates slots by LOC but still stores fixed node widths/heights.

Semantic View Overlap

The semantic layout is better than a raw projection because it clusters and packs points, but at overview zoom the render footprint grows again. A final collision pass over rendered boxes keeps semantic neighborhoods while removing text-on-text collisions.

Recommendations

P0

Introduce a shared footprint service

Move the LOC, zoom, label, badge, and compact sizing logic behind an exported layout-facing API such as getSymbolRenderFootprint(symbol, placement, viewportZoom, context). Use it from buildLayoutGroupContainers, semantic refinement, symbol layout slots, and tests.

P0

Make group bounds footprint-aware

In buildLayoutGroupContainers, replace placement.width ?? getDefaultNodeWidth(node) for symbols with the shared footprint width/height. This fixes the folder overflow without changing agent layout proposals.

P1

Repack custom folders after sizing

For agent/custom layouts, treat group membership as the durable intent and positions as a draft. Add a folder-local repack mode that uses shelf/grid packing by final footprint, with stable ordering by existing row/column and LOC.

P1

Add a final collision pass

After semantic projection/refinement and after custom-group placement, run a deterministic collision resolver over rendered boxes. Keep cluster centroids as anchors, but separate nodes until boxes plus a small label gutter no longer intersect.

P1

Decouple visual importance from physical box growth

Keep LOC prominence, but cap physical growth more aggressively inside groups. Put some importance into a stripe width, halo, minimap color, LOC badge, or z-index instead of only width/height.

P2

Use zoom-level detail policies

At far zoom, show large symbols as readable “summary tiles” and hide secondary badges. At near zoom, allow full labels. This avoids making overview nodes enormous just to keep text legible.
